## 核心原则

1. **需求确认** - 复杂任务先给方案，再动手
2. **进度同步** - 执行中遇到问题及时说
3. **结果明确** - 完成后说清楚做了什么、产出是什么

## 质量标准

- **准确**：不确定的标注出来
- **完整**：不做一半
- **可用**：产出能直接用

- 每周汇总 → 周文档 + 判断是否进长期记忆

## 五文件工作流（复杂项目）

> 来源：Derrick Choi（OpenAI 工程师）的 5 文件管理法
> 成果：AI 25 小时无人值守完成 3 万行代码开发

### 核心思想

```
Context Window = RAM（易失，有限）
Filesystem = Disk（持久，无限）
→ 重要的东西写进文件里。
```

### 五文件结构

| 文件 | 优先级 | 核心功能 | 关键要素 |
|------|--------|----------|----------|
| `docs/prompt.md` | ★★★★★ | 定义项目边界 | 目标/排除项/交付物 |
| `docs/plans.md` | ★★★★★ | 任务拆解与验证 | 1-2 小时里程碑 + 验收命令 |
| `docs/architecture.md` | ★★★ | 技术规范锁定 | 防止架构频繁变更 |
| `docs/implement.md` | ★★★ | 操作手册 | 详细执行步骤 |
| `docs/documentation.md` | ★★ | 进度追踪 | AI 工作日志 |

### 核心机制

**1. 防跑偏（prompt.md）**
- 排除项是核心 — 明确不要什么

**2. 小步快跑（plans.md）**
- 每个里程碑 1-2 小时可完成
- 必须有验收命令

**3. 强制验证**
- 验证不通过，禁止进入下一阶段

### 何时使用

| 场景 | 推荐模式 |
|------|----------|
| 复杂开发项目 | 五文件流 |
| 长期维护项目 | 五文件流 |
| 简单任务 | 不需要 |
