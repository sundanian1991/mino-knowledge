## 9 种汇报场景

| 场景 | 适用情况 | 推荐配置 |
|------|----------|----------|
| **向上汇报** | 向领导/C2 汇报方案 | 直属领导 + 数据驱动 + 7 轮 |
| **周会汇报** | 每周例行工作汇报 | 直属领导 + 结果导向 + 5 轮 |

| **临时汇报** | 临时被叫去汇报 | 直属领导 + 结果导向 + 3 轮 |
| **高危消息汇报** | 坏消息/失误/延期 | 直属领导 + 情绪模拟 + 5 轮 |
| **PRD 评审** | 产品需求文档评审 | 跨部门 + 细节控 + 7 轮 |
| **晋升答辩** | 职级晋升评审 | 评审团 + 质疑型 + 10 轮 |
| **融资路演** | 投资人汇报 | 投资人 + 质疑型 + 10 轮 |
| **跨部门协同** | 需要其他团队配合 | 跨部门领导 + 战略型 + 5 轮 |

**快速选择**：不知道选什么？说"帮我选一个场景"，Claude 会根据你的描述推荐。

## 使用流程

### Step 1: 选择场景

Claude 会问你这几个问题：

1. **汇报场景**：向上汇报 / 周会 / 单独事项 / 临时汇报 / 高危消息 / PRD 评审 / 晋升答辩 / 融资路演 / 跨部门协同
2. **职级关系**：直属领导 / 部门大领导 / 跨部门领导 / 外部评审
3. **管理风格**：数据驱动 / 结果导向 / 细节控 / 战略型 / 质疑型
4. **质询轮次**：快速验证 (3-5 轮) / 标准质询 (7 轮) / 深度挑战 (10+ 轮)

**或者直接说**："我要准备 XX 汇报"，Claude 会自动配置。

### Step 2: 提供方案背景

提供方式：
- 直接粘贴方案文本
- 告诉 Claude 方案文件路径（Claude 会读取）
- 口头描述方案框架

**最少信息**：
1. 方案目标（要解决什么问题）
2. 核心内容（打算怎么做）
3. 预期结果（成功的定义）

### Step 3: 接受质询

Claude 会扮演你的领导，进行连续追问。

**示例（向上汇报）**：
```
AI: Q1/7：你的方案要解决的核心问题是什么？为什么现在是最佳时机？

你：[回答]

AI: Q2/7：你说"效率提升 30%"，这个数据是怎么测算出来的？样本是什么？

你：[回答]

...（继续 7 轮）
```

### Step 4: 查看漏洞清单

质询完成后，Claude 会输出：

1. **漏洞清单** — 暴露的思维盲区表格
2. **高频问题预测** — 领导最可能问的 5 个问题
3. **数据准备清单** — 需要补充的数据和材料
4. **应答策略建议** — 话术和应对策略

### Step 5: 修复方案（可选）

根据漏洞清单优化方案后，可以说：

```
我修改了方案，需要再质询一次。
```

或：

```
只针对刚才的薄弱点再验证一下。
```

## 特殊场景：高危消息汇报

此场景下，AI 会模拟**领导情绪反应**：

- 语气严肃
- 追问责任
- 表达失望

**目的**：让你在高压力环境下练习应对能力。

### 应答策略

**错误示范**：
- "这不是我的责任"
- "我以为不会出问题"
- "我也没办法"

**正确示范**：
- "这是我的责任，我没有..."
- "发现后我第一时间..."
- "目前有 3 个补救方案..."
- "长期来看我们会建立..."

## 最佳实践

### ✅ 建议
1. **方案初稿完成后立即质询** — 越早暴露漏洞，修复成本越低
2. **选择接近真实领导的风格** — 越真实，效果越好
3. **认真对待每一轮回答** — 当真实汇报来应对
4. **质询后必须修复** — 漏洞清单要落实到方案修改
5. **二次验证** — 修复后再质询一次，确认漏洞已修复

### ❌ 避免
1. **为了通过而通过** — 不要糊弄，AI 会追问
2. **只质询不修复** — 漏洞清单要落实到方案修改
3. **风格选择太理想化** — 按真实领导选，不是按"希望领导是这样"选

## 常见问题

**Q: 可以同时模拟多个领导吗？**
A: 建议一次只模拟一个，聚焦深度。如需多视角，可以分多次进行。

**Q: 质询轮次越多越好吗？**
A: 不是。7 轮是最佳平衡点，能覆盖主要维度又不会太累。3 轮用于快速验证，10 轮用于极端重要场合。

**Q: 如果领导风格不好描述怎么办？**
A: 说"帮我选一个"，然后描述你的实际场景（如"互联网大厂晋升答辩"），Claude 会推荐合适的预设。

**Q: 质询过程中可以暂停吗？**
A: 可以。随时说"我先思考一下"或"这个问题我需要查数据"，Claude 会等你。

## 核心理念

**优秀汇报的本质**：不是完美展示，而是建立经得起质疑的逻辑体系。

**填写领导风格参数的过程**：本身就是对管理需求的深度思考。

**低风险试错的价值**：AI 模拟提供安全的"压力测试场"，允许在无后果环境中暴露短板。

---

*在安全环境中暴露的每一个漏洞，都是正式汇报时避开的一个坑。*
