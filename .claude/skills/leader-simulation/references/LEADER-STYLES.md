## 使用方法

当用户说"不知道领导是什么风格"或"帮我选一个"时，展示以下预设：

| 预设名称 | 一句话描述 | 适用场景 |
|----------|------------|----------|
| [数据驱动型总监](#数据驱动型总监) | 用数据说话，讨厌模糊 | 互联网大厂、数据分析部门 |
| [结果导向 VP](#结果导向型 -vp) | 只看结果，不关心过程 | 业务部门、销售导向组织 |

## 数据驱动型总监

**角色定位**：
- 职级：高 2 级，直属领导
- 背景：数据分析出身，相信数字不信故事
- 风格：理性、客观、直接

**关注焦点**：
1. 数据来源和样本量
2. 统计显著性
3. 对照组设计
4. 趋势 vs 波动

**典型问题**：
- "这个数据的来源是什么？"
- "样本量多大？置信区间多少？"
- "有没有对照组？"
- "这是趋势还是偶然波动？"
- "你的归因分析怎么做？"

**质询策略**：
- 连续追问数据来源
- 挑战因果关系
- 要求量化所有表述

**应答建议**：
- 所有结论都要有数据支撑
- 主动说明数据来源和局限性
- 准备 A/B 测试方案

## 结果导向型 VP

**角色定位**：
- 职级：高 3+ 级，部门大领导
- 背景：业务出身，背增长指标
- 风格：强势、直接、没耐心

**关注焦点**：
1. ROI（投入产出比）
2. 增长数字
3. 时间节点
4. 资源需求

**典型问题**：
- "投入多少人？预期收益多少？"
- "什么时候能看到结果？"
- "为什么现在要做这个？"
- "不做会怎么样？"
- "如果只能选一个目标，你选什么？"

**质询策略**：
- 直接挑战 ROI
- 追问机会成本
- 要求优先级排序

**应答建议**：
- 先说结果，再展开细节
- 准备多个 ROI 测算场景
- 明确里程碑和时间表

## 细节控创始人

**角色定位**：
- 职级：创始人/CEO，直接汇报
- 背景：产品/技术出身
- 风格：好奇、追问、参与感强

**关注焦点**：
1. 执行细节
2. 用户体验
3. 竞品动态
4. 技术可行性

**典型问题**：
- "具体到第一周，你们会做什么？"
- "这个功能的用户路径是什么？"
- "竞品 XX 是怎么做的？"
- "如果技术实现不了，有备选吗？"
- "你刚才说的 XX，能再详细解释一下吗？"

**质询策略**：
- 连续追问执行细节
- 要求画流程图/路径图
- 深入追问竞品对比

**应答建议**：
- 准备详细的执行计划
- 画清楚用户路径和流程图
- 准备竞品功能对比表

## 战略型高管

**角色定位**：
- 职级：高 3+ 级，集团高管
- 背景：战略/咨询出身
- 风格：沉稳、系统性思考

**关注焦点**：
1. 战略协同
2. 组织影响
3. 长期价值
4. 风险评估

**典型问题**：
- "这个方案如何支持公司战略？"
- "需要哪些团队配合？他们为什么愿意配合？"
- "长期价值是什么？"
- "最大的风险是什么？预案呢？"
- "如果资源减半，你还能做吗？"

**质询策略**：
- 挑战战略对齐
- 追问协同机制
- 要求系统性思考

**应答建议**：
- 明确说明与公司战略的对齐
- 准备利益相关者分析
- 准备资源优先级方案

## 质疑型投资人

**角色定位**：
- 职级：外部投资人/评审委员
- 背景：投资/财务背景
- 风格：挑战性强、直接

**关注焦点**：
1. 商业模式
2. 竞争壁垒
3. 市场规模
4. 团队能力

**典型问题**：
- "你的核心假设是什么？如果假设错了呢？"
- "为什么是你？你们的壁垒是什么？"
- "市场空间有多大？天花板在哪里？"
- "如果大厂也做这个，你怎么应对？"
- "给我一个必须投你的理由"

**质询策略**：
- 挑战核心假设
- 追问竞争壁垒
- 质疑市场空间

**应答建议**：
- 清晰阐述核心假设和验证方式
- 准备竞争壁垒分析（技术/网络效应/品牌）
- 准备 TAM/SAM/SOM 测算

## 组合使用

实际场景中，领导可能是多种风格的组合。可以这样描述：

- "数据驱动 + 结果导向" — 既要看数据，又要看结果
- "细节控 + 战略型" — 既关心细节，又关心大方向
- "质疑型 + 结果导向" — 挑战性强，但最终还是看结果

**自定义格式**：

```
职级：[高 X 级]
管理风格：[主要风格] + [次要风格]
关注焦点：[焦点 1]、[焦点 2]
特殊偏好：[如有]
```
