## 向上汇报

### 适用情况
- 向直属领导（C2/C3）汇报新方案
- 需要领导审批或资源支持
- 正式会议场合

### 典型场景
```
"领导，我想汇报一下新的供应商定价委员会机制方案，需要您审批。"
```

### 领导关注点
1. **方案必要性**：为什么现在要做？不做会怎样？
2. **投入产出比**：需要多少资源？预期收益？
3. **风险可控性**：有什么风险？预案是什么？
4. **战略对齐**：与部门/公司目标是否一致？

### 质询模板（7 轮）

```
Q1: 你的方案要解决的核心问题是什么？为什么现在是最佳时机？

Q2: 你说"目前定价调整缺乏约束"，有具体数据吗？发生了多少次？影响多大？

Q3: 这个方案需要投入多少人力？供应商管理岗之外还需要谁参与？

Q4: 如果供应商反弹比预期大，你有什么预案？

Q5: 你说"单次降幅≤30%"，这个阈值是怎么定的？为什么不是 20% 或 40%？

Q6: 这个机制和现有的供应商管理流程是什么关系？是替代还是补充？

Q7: 如果我是领导，听完你的汇报，我最可能挑战的点是什么？
```

### 应答策略
- **数据先行**：所有论断都要有数据支撑
- **预案充分**：每个风险至少准备 2 个预案
- **阈值有依据**：关键数字要有测算过程

## 周会汇报

### 适用情况
- 每周例行工作汇报
- 进展同步 + 问题暴露
- 需要领导协调资源

### 典型场景
```
"这周主要推进了三件事：定价委员会机制落地、供应商沟通会、数据分析..."
```

### 领导关注点
1. **进展是否正常**：按计划推进吗？
2. **有无风险**：遇到什么问题？需要我做什么？
3. **工作价值**：做的事情对目标有贡献吗？

### 质询模板（5 轮）

```
Q1: 这周最重要的进展是什么？用一句话说清楚。

Q2: 你说"进展顺利"，具体完成了什么？有什么可量化的成果？

Q3: 遇到什么困难了吗？需要我协调什么资源？

Q4: 下周的计划是什么？优先级怎么排的？

Q5: 你现在做的这些事，对部门 Q1 目标的贡献是什么？
```

### 应答策略
- **结论先行**：先说最重要的进展
- **量化成果**：用数字说话
- **主动暴露风险**：不要等领导问
- **明确资源请求**：需要什么说清楚

## 单独事项汇报

### 适用情况
- 专项问题深度汇报
- 突发事件处理
- 需要领导决策

### 典型场景
```
"领导，有个紧急情况需要汇报。XX 供应商对降价方案有强烈反弹..."
```

### 领导关注点
1. **问题定位**：到底是什么问题？
2. **根因分析**：为什么会发生？
3. **解决方案**：有几个选项？各有什么利弊？
4. **是否复发**：如何防止再发生？

### 质询模板（7 轮）

```
Q1: 用一句话说清楚，发生了什么问题？

Q2: 这个问题的根因是什么？你如何确定的？

Q3: 你有什么解决方案？至少给我 3 个选项。

Q4: 你推荐哪个方案？为什么？

Q5: 这个方案最大的风险是什么？

Q6: 如何防止类似问题再次发生？

Q7: 如果让你重新来一次，你会有什么不同的做法？
```

### 应答策略
- **问题清晰**：用事实描述，不带情绪
- **根因深入**：至少问 5 个为什么
- **方案对比**：提供多个选项 + 推荐
- **长期机制**：不仅解决当下，还要防止复发

## 临时汇报

### 适用情况
- 突然被叫去领导办公室
- 电梯里遇到领导被问到
- 临时被要求汇报进展

### 典型场景
```
"来我办公室一下，说说那个项目的进展。"
```

### 领导关注点
1. **核心事实**：发生了什么？
2. **紧急程度**：需要我立刻处理吗？
3. **建议方案**：你建议怎么做？

### 质询模板（3 轮快速）

```
Q1: 用 30 秒说清楚，现在是什么情况？

Q2: 需要我做什么决策？或者给什么支持？

Q3: 如果我不干预，你能搞定吗？多长时间？
```

### 应答策略
- **电梯演讲**：30 秒内说清楚核心
- **明确请求**：需要领导做什么说清楚
- **承担责任**：展现你能搞定的信心

## 高危消息汇报

### 适用情况
- 项目延期/失败
- 重大失误
- 供应商强烈反弹
- 客户投诉

### 典型场景
```
"领导，有个 bad news 需要汇报。我们可能无法按期交付了..."
```

### 领导关注点
1. **态度**：你是否认识到严重性？
2. **补救方案**：你打算怎么解决？
3. **是否隐瞒**：还有没有别的我不知道的？
4. **预防措施**：如何防止再发生？

### 质询模板（5 轮 + 情绪模拟）

```
Q1: [语气严肃] 说吧，怎么回事？

Q2: 这个问题你为什么不早说？什么时候发现的？

Q3: 现在最坏的情况是什么？

Q4: 你有什么补救方案？什么时候能解决？

Q5: 如何保证类似问题不再发生？
```

### AI 特殊功能：情绪模拟

在此场景下，AI 会模拟领导的情绪反应：
- 语气严肃
- 追问责任
- 表达失望

**目的**：让你在高压力环境下练习应对能力。

### 应答策略
- **主动承担**：不要推卸责任
- **及时暴露**：强调"第一时间汇报"
- **补救优先**：重点说解决方案
- **长期预防**：说明如何防止复发

**错误示范**：
- "这不是我的责任"
- "我以为不会出问题"
- "我也没办法"

**正确示范**：
- "这是我的责任，我没有..."
- "发现后我第一时间..."
- "目前有 3 个补救方案..."
- "长期来看我们会建立..."

## PRD 评审

### 适用情况
- 产品需求文档评审
- 技术方案评审
- 跨部门方案对齐

### 典型场景
```
"这是新版本的供应商管理系统 PRD，请大家评审。"
```

### 关注点
1. **需求价值**：为什么要做？用户是谁？
2. **技术可行性**：能实现吗？成本多少？
3. **边界完整性**：异常情况考虑了吗？
4. **优先级**：为什么现在做？

### 质询模板（7 轮）

```
Q1: 这个需求解决的用户痛点是什么？有多少用户有这个痛点？

Q2: 如果不做这个功能，会有什么影响？

Q3: 技术实现方案是什么？有没有技术风险？

Q4: 开发周期多长？需要多少人力？

Q5: 异常流程考虑了吗？比如 XX 情况怎么处理？

Q6: 和现有系统的关系是什么？会影响其他功能吗？

Q7: 如何衡量这个功能成功了？指标是什么？
```

### 应答策略
- **用户视角**：从用户价值出发
- **数据支撑**：需求优先级有数据
- **技术可行**：和技术团队确认过
- **边界完整**：异常流程考虑周全

## 晋升答辩

### 适用情况
- 职级晋升评审
- 年度绩效评估
- 人才盘点

### 典型场景
```
"我申请晋升到 C3，以下是我的工作成果..."
```

### 评审关注点
1. **价值贡献**：你创造了什么价值？
2. **能力成长**：相比之前有什么提升？
3. **方法论沉淀**：能复制给其他人吗？
4. **潜力**：还能承担更大责任吗？

### 质询模板（10 轮）

```
Q1: 用一句话说清楚，你最大的价值贡献是什么？

Q2: 这个成果中，你个人的贡献占比是多少？

Q3: 遇到什么困难？如何克服的？

Q4: 你的工作和 C2/C3 的要求对齐在哪里？

Q5: 如果让你重新做一次，你会有什么不同的做法？

Q6: 你的方法论是什么？能复制给其他人吗？

Q7: 你觉得自己还有什么不足？

Q8: 晋升后，你计划做什么？

Q9: 如果资源减半，你还能达成目标吗？

Q10: 给我一个必须晋升你的理由。
```

### 应答策略
- **价值量化**：用数字证明贡献
- **突出个人**：区分团队成果和个人贡献
- **方法论**：展现可复制的能力
- **自我认知**：清楚自己的不足和改进计划

## 融资路演

### 适用情况
- 投资人汇报
- 董事会汇报
- 战略合作洽谈

### 典型场景
```
"我们是 XX 公司，致力于解决 XX 问题，这是我们的商业模式..."
```

### 投资人关注点
1. **市场空间**：天花板有多高？
2. **商业模式**：怎么赚钱？
3. **竞争壁垒**：为什么是你？别人进不来吗？
4. **团队能力**：为什么这个团队能做成？
5. **退出机制**：我怎么退出？

### 质询模板（10 轮）

```
Q1: 用一句话说清楚，你们是做什么的？

Q2: 市场空间有多大？TAM/SAM/SOM 分别是多少？

Q3: 你们的商业模式是什么？怎么赚钱？

Q4: 核心竞争力是什么？壁垒在哪里？

Q5: 如果大厂也做这个，你怎么应对？

Q6: 目前的进展如何？关键指标是什么？

Q7: 为什么现在是最佳时机？

Q8: 这轮融资计划用在哪里？预期达成什么目标？

Q9: 你们团队的独特优势是什么？

Q10: 给我一个必须投你的理由。
```

### 应答策略
- **故事清晰**：一句话能说清楚
- **数据扎实**：关键指标要过硬
- **壁垒明确**：说清楚为什么别人进不来
- **团队匹配**：为什么这个团队能做成

## 跨部门协同

### 适用情况
- 需要其他团队配合
- 资源协调
- 流程对接

### 典型场景
```
"这个项目需要你们团队提供一些支持..."
```

### 对方关注点
1. **为什么配合你**：对我有什么好处？
2. **资源投入**：需要我投入多少？
3. **优先级**：比我自己的事重要吗？
4. **责任边界**：出了事谁负责？

### 质询模板（5 轮）

```
Q1: 你需要我们做什么？具体一点。

Q2: 这个项目对你们的核心价值是什么？对我们呢？

Q3: 需要我们投入多少人力？多长时间？

Q4: 如果配合你，我们能得到什么？

Q5: 如果出了问题，谁负责？
```

### 应答策略
- **利益对齐**：先说对对方有什么好处
- **投入清晰**：明确需要多少资源
- **责任明确**：责任边界说清楚
- **资源交换**：你能给对方什么
