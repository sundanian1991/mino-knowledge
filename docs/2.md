二、关键文件实施指南
1. prompt.md：精准定义需求边界
目标：明确"要造什么"（如"开发带实时协作功能的设计工具"）
排除项：明确"不要什么"（如"不需要移动端适配"）——此为核心防跑偏机制
交付物：列出必须包含的功能模块（如"图层管理/版本历史/React代码导出"）
完成标准：可量化指标（如"通过所有单元测试，无console错误"）
2. plans.md：任务拆解与质量控制
里程碑粒度：控制在1-2小时可完成的最小单元
验收机制：每个节点必须包含可执行验证命令（如npm run test/npm run lint）
硬性规则：验证不通过禁止进入下一阶段，强制AI自我修正
🚀 实战应用与成果展示
一、25小时开发成果
AI自主完成的设计工具具备以下核心功能：
画布编辑系统：支持图形绘制、文本插入、图层管理
实时协作：多人同步编辑与 presence 状态显示
版本控制：完整历史记录与回溯功能
代码导出：直接生成可复用的React组件代码
二、三步落地法
准备阶段：创建docs目录，完成prompt.md和plans.md（核心必做）
启动指令："先读docs/下所有文件，按plans.md顺序执行，每步跑验证命令"
监控模式：AI自主运行，开发者仅需定期查看documentation.md进度
💡 关键洞察与补充细节
工具普适性：方法适用于Claude Code、Cursor、Copilot等主流AI编程工具
开源资源：Derrick的5个文件模板已开源（GitHub搜索：design-desk，docs/目录可直接复用）
效率提升原理：通过文档化解决AI的"短期记忆"缺陷，将人类抽象需求转化为机器可执行的具体指令链
避坑指南：80%无效开发时间源于目标不明确和缺乏验证机制，plans.md的"小里程碑+强验证"是核心解决方案
