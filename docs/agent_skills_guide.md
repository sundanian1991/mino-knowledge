# Agent 技能配置与使用指南

## 前置条件
- 技能安装：需先安装 self-improving-agent 技能包。
- 文件修改：在 AGENTS.md 配置文件中添加指定指令代码段。

## 技能安装
- 使用 Skills CLI 安装 self-improving-agent：
  - 命令示例：`npx skills add openclaw/skills/tree/main/skills/dennis-da-menace/agent-memory/SKILL.md`（若该仓库提供安装入口，请按技能说明执行安装）
  - 安装后验证：运行 `skills check` 确认版本与依赖。

## AGENTS.md 配置
- 在 AGENTS.md 中添加以下指令代码段（示例）：

```markdown
## 指令模糊处理
“每一次我的指令你觉得模棱两可，你都要指出来所有的可能性，并且让我做选择以后 再开始行动”

## 复盘机制激活
“每一次我说复盘，你都要调用 自我提升 skill 并且把 review 记录到对应的记忆文档里面”
```

- 配置说明：
  - 将上述两段指令置于 AGENTS.md 的“指令与行为规范”章节，确保在每次会话开始时加载。
  - 若存在多版本 AGENTS.md，请同步更新所有副本。

## 触发场景
- 指令模糊处理：当用户指令存在多种合理解释或潜在歧义时触发。
- 复盘机制激活：当用户明确说“复盘”或在工作流结束后主动要求复盘时触发。

## 具体指令内容
- 指令模糊处理：
  - 识别歧义点（例如：范围、优先级、输出格式、时间窗口）。
  - 列出所有合理可能性与各自影响。
  - 请求用户选择或补充约束。
- 复盘机制激活：
  - 调用 self-improving-agent 的 review 流程。
  - 生成复盘要点（成功点、问题点、改进项、行动项）。
  - 将复盘记录写入记忆文档（按日期归档）。

## 功能说明
- 指令模糊处理：增强指令理解准确性，避免执行偏差，提高任务成功率。
- 复盘机制激活：实现自动复盘与学习成果固化，形成持续改进闭环。

## 指令模糊处理（详细）
- 触发条件：检测到指令存在语义歧义、范围不清、优先级不明、输出格式不确定等。
- 处理步骤：
  1) 识别歧义点并分类（范围/优先级/格式/时间/依赖）。
  2) 为每个可能性提供简短示例与影响评估。
  3) 向用户提供选择项（A/B/C 或多选），等待确认。
  4) 记录选择与约束，作为后续执行的上下文。

## 复盘机制（详细）
- 触发条件：用户明确说“复盘”，或在任务结束后自动触发（可配置）。
- 处理步骤：
  1) 调用 self-improving-agent 的 review 子流程。
  2) 汇总本次任务的关键决策、数据与结果。
  3) 生成复盘报告（成功点、问题点、根因分析、改进项、行动项）。
  4) 将复盘报告写入记忆文档（按日期归档），并更新知识库索引。

## 安装与验证清单
- 安装 self-improving-agent 技能包。
- 更新 AGENTS.md 并重启会话或重新加载配置。
- 测试指令模糊处理：输入含歧义的指令，确认代理会列出可能性并等待选择。
- 测试复盘机制：输入“复盘”，确认生成复盘报告并写入记忆文档。
