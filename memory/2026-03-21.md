# 2026-03-21 记忆条目：OpenClaw-bot-review 仪表盘接入方案

## 背景
- 用户分享“养🦐日记-Day 6”，记录 JoyClaw 接入 OpenClaw-bot-review 仪表盘的实践，包括路径报错与自愈、多 Agent 状态监控、像素办公室、Token 用量统计、任务失败告警。

## 关键经验
- 路径适配：开源工具默认读取 ~/.openclaw，而 JoyClaw 根路径为 ~/.joyclaw；通过设置 OPENCLAW_HOME=/Users/liuzhenning/.joyclaw 或指定配置文件路径解决 ENOENT 错误。
- 部署流程：Node.js ≥ 22、pnpm、git；克隆仓库、pnpm install、构建与启动；按项目 README 执行启动命令。
- Dashboard 接入：默认 http://127.0.0.1:18789/ 或子路径；前端通过 WebSocket 直连 Gateway；远程访问建议 SSH 隧道或 Tailscale。
- 功能亮点：全量状态监控、模型与 Token 透明化、像素办公室互动、任务失败告警与定位。
- 安全与运维：避免在 URL 泄露 token；优先 SecretRef/环境变量；HTTPS + 设备配对；定期更新与日志排查。

## 可执行命令（示例）
- 安装 CLI（可选）：curl -fsSL https://openclaw.ai/install.sh | bash 或 npm install -g openclaw@latest
- 克隆与安装：git clone https://github.com/liuzhenning/OpenClaw-bot-review.git && cd OpenClaw-bot-review && pnpm install
- 路径适配：export OPENCLAW_HOME=/Users/liuzhenning/.joyclaw
- 启动：按项目 README 的启动脚本执行（如 pnpm start）
