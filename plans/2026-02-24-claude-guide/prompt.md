# Prompt - 项目定义

> **状态**：待用户确认
>
> **核心**：定义边界，防止跑偏

---

## AI 必读

**编辑完此文件后必须**：
1. 向用户展示 `prompt.md` 和 `plans.md` 草案
2. 使用 AskUserQuestion 或明确询问"需求边界是否准确？"
3. **等待用户确认后才能执行**

---

## 目标

开发一个**Claude Code 交互式学习网页**，能够让用户通过可视化、交互式的方式深入学习 Claude Code 的七大组件（CLAUDE.md、Commands、Hooks、SubAgents、Skills、MCP Servers、Plugins）。

---

## 排除项

**核心防跑偏机制** — 明确不要什么：

- [x] 不需要后端服务（纯前端静态页面）
- [x] 不需要用户登录/账号系统
- [x] 不需要复杂的游戏化机制（积分/排行榜）
- [x] 暂不实现 PWA/离线功能
- [x] 不需要深色/浅色主题切换（专注一种审美）

---

## 交付物

必须包含的功能模块：

- [ ] **七大组件卡片展示** - 每个组件一个精美卡片，含简介、痛点、解决方案
- [ ] **交互式代码预览** - Hooks、Commands 等的配置代码可交互查看
- [ ] **渐进式学习路径** - 按"基础→进阶→实战"层级组织内容
- [ ] **组合案例展示** - 展示 1+1>2 的组件组合效果
- [ ] **立即可做清单** - 底部的实战建议 checklist
- [ ] **响应式设计** - 桌面端和移动端都有良好体验

---

## 完成标准

可量化的验收指标：

- [ ] 所有七大组件内容完整呈现
- [ ] 至少 3 种交互效果（hover、点击、滚动触发）
- [ ] 无 console 错误
- [ ] Lighthouse 性能评分 > 80
- [ ] 页面加载时间 < 2s

---

## 约束条件

- **技术栈**：HTML5 + Tailwind CSS + 原生 JavaScript（无框架依赖）
- **设计**：Clean > Minimalist > Modern，p-6/p-8 间距，rounded-xl 圆角
- **位置**：`workspace/claude-guide/index.html`
- **单文件**：所有代码封装在一个 HTML 文件中
